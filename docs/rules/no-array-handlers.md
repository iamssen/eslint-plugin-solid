# no-array-handlers

## 1. 규칙이 존재하는 이유 (Solid.js 1.0 기준)

Solid.js는 이벤트 핸들러(예: `onClick`, `onInput`)에 특별한 배열 문법을 지원합니다. 핸들러 함수와 함께 인자를 배열 형태로 전달하면, 렌더링 시점에 추가적인 클로저(Closure) 함수를 생성하지 않고도 이벤트를 처리할 수 있습니다. 

예시 (Solid.js 지원 문법):
```jsx
<button onClick={[handleClick, 123]}>클릭</button>
```

하지만 일부 개발팀이나 프로젝트에서는 위와 같은 배열 문법이 표준적인 자바스크립트/리액트 패턴과 달라 가독성을 떨어뜨린다고 판단할 수 있습니다. 또한 타입스크립트 환경에서 배열 문법을 사용할 때 타입 추론이 제대로 되지 않거나 복잡해지는 문제가 발생할 수 있습니다.

이 규칙(`no-array-handlers`)은 이벤트 핸들러에 배열 문법을 사용하는 것을 금지하고, 대신 모든 자바스크립트 개발자에게 익숙한 표준 화살표 함수(인라인 클로저) 형태로 이벤트를 작성하도록 강제하기 위해 존재합니다.

## 2. Solid.js 2.0에서의 변경 여부

Solid.js 2.0에서는 컴파일러와 런타임이 개선되면서, 굳이 클로저 생성을 피하기 위해 배열 문법을 사용할 필요성이 더욱 줄어듭니다. 오히려 화살표 함수를 사용하는 것이 훨씬 더 표준적이고 컴파일 타임 최적화에도 유리하게 작용할 수 있습니다. 따라서 이 규칙을 통해 코드를 화살표 함수 방식으로 통일해두면, Solid.js 2.0으로의 전환에 더 유리(Future-proof)할 수 있습니다.

## 3. 규칙의 이해를 돕기 위한 추가 설명 및 예제

### ❌ 잘못된 사용 예 (배열 문법 사용)

아래는 Solid.js의 배열 문법을 사용하여 이벤트 핸들러와 매개변수를 전달하는 경우입니다. 이 규칙이 활성화되어 있으면 ESLint 에러가 발생합니다.

```jsx
// ❌ 배열 문법 사용 금지
function MyComponent() {
  const handleClick = (id, event) => console.log(id);

  return (
    // [핸들러_함수, 첫_번째_인자] 형태를 사용함
    <button onClick={[handleClick, 123]}>
      클릭하세요
    </button>
  );
}
```

### 🟢 올바른 사용 예 (화살표 함수 사용)

배열 문법 대신 자바스크립트 표준 화살표 함수를 사용하여 이벤트 객체와 필요한 데이터를 명시적으로 전달합니다.

```jsx
// 🟢 표준 화살표 함수 사용
function MyComponent() {
  const handleClick = (id, event) => console.log(id);

  return (
    <button onClick={(e) => handleClick(123, e)}>
      클릭하세요
    </button>
  );
}
```

### 시각적 요약

- **배열 문법 (`onClick={[fn, data]}`)**: Solid 전용 문법. 초보자가 보기에 어떤 값이 먼저 들어가는지 헷갈릴 수 있고, 타입스크립트 지원에 한계가 있음.
- **화살표 함수 (`onClick={(e) => fn(data, e)}`)**: 자바스크립트 표준. 코드가 명확하고 직관적이며 어떠한 환경에서도 완벽히 호환됨.
