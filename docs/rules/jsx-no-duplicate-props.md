# jsx-no-duplicate-props

## 1. 규칙이 존재하는 이유 (Solid.js 1.0 기준)

JSX(JavaScript XML) 문법을 사용하여 컴포넌트나 HTML 엘리먼트를 작성할 때, 동일한 이름의 속성(prop)을 두 번 이상 중복해서 작성하는 실수를 방지하기 위해 존재합니다.

JSX 컴파일러는 중복된 속성이 있을 경우 대체로 가장 마지막에 선언된 속성의 값을 사용하지만, 이는 개발자의 실수(오타 또는 복사/붙여넣기 오류)일 확률이 높습니다. 의도하지 않게 값이 덮어씌워져 발생하는 버그를 미연에 방지하기 위해, 하나의 태그 내에서 동일한 속성명은 단 한 번만 사용하도록 강제합니다.

## 2. Solid.js 2.0에서의 변경 여부

이 규칙은 Solid.js 프레임워크 자체의 반응성 시스템과는 무관하게 JSX 문법의 린팅(Linting) 영역에 속합니다. 
Solid.js 2.0에서도 여전히 동일한 JSX 문법을 사용하므로, 이 규칙은 버전과 상관없이 항상 유효하고 변함없이 적용됩니다.

## 3. 규칙의 이해를 돕기 위한 추가 설명 및 예제

### ❌ 잘못된 사용 예 (속성 중복)

아래 예시들에서는 `id` 속성이나 `name` 속성이 한 요소 안에서 여러 번 정의되었습니다.

```jsx
// ❌ id 속성이 두 번 쓰임 (뒤에 있는 id="two"가 적용됨)
<div id="one" id="two" />

// ❌ name 속성이 두 번 쓰임
<Hello name="John" name="Doe" />
```

### 🟢 올바른 사용 예 (단일 속성)

모든 속성은 각 요소 내에서 한 번만 고유하게 선언되어야 합니다.

```jsx
// 🟢 고유한 속성만 존재
<div id="one" class="container" />

<Hello firstName="John" lastName="Doe" />
```

### 팁

- 만약 동적으로 속성을 덮어씌워야 하는 의도적인 상황이라면, 속성을 직접 중복 작성하는 대신 객체 전개 연산자(`...spread`)와 `mergeProps` 등을 조합하여 사용하는 것이 안전하고 명확합니다.
